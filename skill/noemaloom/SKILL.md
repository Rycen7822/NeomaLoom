# NoemaLoom

Use NoemaLoom for repository-wide modification localization, impact tracing, and post-edit coverage verification.

NoemaLoom locates and verifies spans. Codex or Hermes edits files with native tools. NoemaLoom does not write project source files, global agent config, Git hooks, or agent caches.

## Routing

Read only the relevant reference for the current task:

- Repository localization: `references/repository_locator.md`
- Markdown or docs updates: `references/markdown_update.md`
- Code API changes and impact tracing: `references/code_change_impact.md`
- Multi-document synchronization: `references/multi_doc_sync.md`
- Post-edit coverage checks: `references/coverage_verification.md`
- Context compression recovery: `references/compression_recovery.md`

Use NoemaLoom MCP tools for locating, tracing, reading bounded spans, and verification. Use native Codex or Hermes editing commands after NoemaLoom identifies spans.
